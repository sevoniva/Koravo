# 快速开始

## 1. 启动依赖

在项目根目录执行：

```bash
docker compose up -d postgres redis minio
```

看到 PostgreSQL、Redis、MinIO 容器启动后继续。

## 2. 启动后端和前端

后端启动后访问 `http://localhost:8080/swagger-ui.html` 可查看接口。

前端启动后打开 `http://localhost:8000`。

## 3. 检查流程配置

进入「流程模型」，确认「协同审批流程」已部署，且配置路径显示「设计、发布、表单、发起、追踪」。

如果环境为空，管理员按以下顺序完成配置：

1. 在「流程模型」创建或导入协同审批流程。
2. 在「表单管理」创建业务申请表，发布后确认使用路径。
3. 在「表单绑定」绑定发起表单和多人会签任务表单。
4. 回到「流程模型」校验并发布。

## 4. 发起业务流程

进入「发起流程」。

选择「协同审批流程」，填写事项名称、事项说明、期望结果，并按角色或人员选择审批人，点击「发起」。申请人和所属部门由当前登录成员自动带出。

## 5. 处理待办

进入「组织权限」确认审批人成员已启用，再进入「我的待办」打开「多人会签」待办。

填写审批结论和处理意见，点击「完成任务」。

每个审批人都处理完成后，流程结束。

## 6. 查看结果

进入「我的申请」打开申请详情。运维人员可进入「流程实例」查看运行实例。

应能看到：

- 实例状态
- 审批进度
- 当前任务
- 各审批人的处理意见
- 表单快照
- 执行轨迹
- 审计记录

管理员进入「审计日志」可查看关键操作记录。
