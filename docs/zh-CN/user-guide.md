# 用户指南

## 首页

首页显示服务状态、流程模型、运行实例、待办任务、连接器调用和最近操作。

点击「创建流程」进入流程设计器，从流程建模开始配置真实业务流程。

## 业务流程

建议按以下顺序跑通一条多人验收流程：

1. 进入「流程模型」，确认「多人验收流程」已通过发布检查并部署。
2. 进入「表单管理」，配置验收申请表字段和校验规则。
3. 进入「表单绑定」，把启动表单、业务验收和财务验收任务绑定到表单版本。
4. 进入「流程模型」，校验并部署流程。
5. 进入「发起流程」，提交验收申请。
6. 进入「我的待办」，由业务处理人和财务复核人分别处理并行待办。
7. 回到「流程实例」查看进度、处理意见、表单快照和审计记录。

「开发资产检查」仅用于开发期补齐缺失流程资产，不作为日常业务入口。

## 系统设置

查看当前登录成员、组织、职责和最近一次请求追踪号。前端会随 API 请求发送当前平台会话上下文，包括：

- `X-Tenant-Id`：当前组织
- `X-User-Id`：当前成员
- `X-User-Role`：当前职责
- `X-Request-Id`：可选追踪号，不填则后端自动生成

## 审计日志

进入「审计日志」，可按用户、动作、资源类型、资源 ID、请求 ID 和时间查询。

关键操作会记录审计，包括流程模型保存、流程部署、流程启动、任务完成、表单保存、表单绑定、数据源测试和连接器调用。
